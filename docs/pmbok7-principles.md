# PMBOK 第七版核心原则 — GameForgeHub 设计依据

## 范式转变：从过程导向到原则导向

第七版（2021）是 PMBOK 自1987年以来最深刻的转变：
- 从 **5大过程组 + 10大知识领域** → **12条原则 + 8个绩效域**
- 从 **ITTO（输入-工具-输出）** → **基于原则的指导性框架**
- 从 **关注可交付物** → **关注成果和价值交付**
- 从 **规定性** → **指导性**，适配所有开发方法

## 12条项目管理原则（摘要）

1. **管家精神** — 以正直、关心和可信的态度管理项目
2. **协作团队** — 协同工作的团队比独自工作的个人更有效率
3. **干系人参与** — 深入参与超越简单沟通
4. **聚焦价值** — 价值是项目成功的最终指标，持续评估与商业目标的一致性
5. **系统思维** — 从整体角度识别项目内外的动态交互
6. **领导力** — 任何成员都可展现领导力，不等于职权
7. **裁剪** — 使用"刚好够"的过程实现预期成果
8. **质量** — 预防优于检测，尽早发现问题
9. **驾驭复杂性** — 复杂性无法控制，但可调整活动应对
10. **优化风险** — 风险包括机会和威胁，持续评估
11. **适应性和韧性** — 较短反馈循环、持续学习、小规模实验
12. **驱动变革** — 采用激励型而非强制型策略

## 裁剪原则 — 小团队做减法

核心理念："既可以规模很大也可以很小，既可以非常严密也可以是轻量级。"

### 可裁剪的维度
- 生命周期和开发方法
- 过程（增加/修改/删除/混合）
- 参与方式（角色、沟通）
- 工具（匹配项目规模）
- 文档（"刚好够"的工件）

### 小团队裁剪建议
| 问题 | 对策 |
|------|------|
| 审批延误 | 简化决策流程，减少授权层级 |
| 在制品过多 | 看板可视化，识别瓶颈 |
| 缺乏可视性 | 收集、分析、分享适当的测量数据 |
| 干系人未参与 | 评估反馈循环是否有效 |

### 裁剪三问
- **赋能：** 是否信任和支持团队自主制定决策？
- **信任：** 是否高度相信团队有能力交付成果？
- **文化：** 组织文化是赋能型还是命令检查型？

## 适应型方法核心特征

- 迭代缩短（1-2周），每迭代结束展示成果
- 待办事项列表驱动工作范围
- 团队积极参与每个迭代规划
- 适用条件：高创新、需求不确定、范围将演变、可增量交付

## 团队自组织与赋能

- 自组织团队中不指定项目经理，由团队成员轮流充当引导者
- 服务型领导力：消除障碍、避免分心、鼓励发展
- 自主、专精和目的是三种内在动机（Daniel Pink 模型）
- 分布式领导力：高绩效项目中多名成员表现出领导力

## 度量指标 — 该保留什么

### 推荐保留的核心指标
- 交付偏差（计划 vs 实际）
- 完成功能数 / 待办总数
- 缺陷率
- 资源利用率
- 团队情绪（情绪图/NPS）
- 燃烧率（速度/故事点）

### 必须警惕的陷阱
- **虚荣指标** — 看似有用但不推动决策
- **霍桑效应** — 测量本身影响被测行为
- **确认偏见** — 只寻找支持既有观点的数据

### 度量的根本目的
只测量和报告能够：**使团队学习、推动决策、改进绩效、避免问题**的信息。

## 在 GameForgeHub 中的落地

| PMBOK 7 原则 | GameForgeHub 实现 |
|---|---|
| 裁剪 / "刚好够" | GitHub 为唯一真相源，不重复存储 |
| 聚焦价值 | 积分系统衡量影响力而非工时 |
| 协作团队 / 自组织 | 任务自主领取，无人强制分配 |
| 适应性和韧性 | SSE 实时反馈，短迭代可见进度 |
| 系统思维 | Quest Board + Idea Arena + Roadmap 三域联动 |
| 领导力 / 赋能 | 分布式积分，任何人可提出想法并被采纳 |
| 度量 | 5-7 个核心指标，避免虚荣指标 |
| 质量 | WIP 限制防止过载，分支健康预警 |
| 驱动变革 | 游戏化激励（积分/成就）而非强制 |
